# Zara Stock Monitor

## Контекст
Мониторинг наличия размеров на Zara с уведомлениями в Telegram.
Запуск через Docker Compose.

## Текущий статус
- Антибот-проблема оказалась мнимой: старый эндпоинт указывал на несуществующий
  zara.ru (404). Рабочий эндпоинт — `itxrest/4/catalog/store/{store_id}/product/id/{id}`,
  отвечает 200 без каких-либо кук (в т.ч. без Akamai bot-manager кук) —
  подтверждено curl и прогоном под нагрузкой (10 запросов подряд).
- Товары теперь добавляются прямо в Telegram: `/add <артикул>` → бот
  показывает список размеров → отвечаешь номером/названием → товар
  сохраняется в `data/products.json` (docker volume) и мониторится.
  Команды: `/add`, `/remove`, `/list`.
- store_id/locale фиксированы в `.env` на весь сайт, per-product сниффинг
  не нужен — `product_id` берётся прямо из URL товара.

## Возможные следующие шаги (не сделано)
- Счётчик подряд идущих ошибок и алерт в Telegram при деградации API
  (было в старой версии скрипта, убрано при переходе на мульти-товарный режим).
- Ограничение на количество отслеживаемых товаров/размеров, если список
  вырастет и один цикл опроса станет слишком длинным.
